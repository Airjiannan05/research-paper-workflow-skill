# Submission Checklist

## Novelty and positioning

- [ ] Main contribution is clearly distinguishable from closest prior work.
- [ ] Related Work compares mechanisms, not only topics.
- [ ] Claims are scoped and falsifiable.

## Experiments

- [ ] Main claims map to experiments.
- [ ] Strong baselines are included or omissions are justified.
- [ ] Ablations test each proposed component.
- [ ] Variance/sensitivity/failure cases are reported when relevant.
- [ ] Compute, data, and implementation details are sufficient.

## Integrity

- [ ] Every factual claim has citation support.
- [ ] Every empirical claim has experiment provenance.
- [ ] No fabricated references, metrics, or results.
- [ ] Limitations are specific and honest.

## Formatting and policy

- [ ] Target venue format followed.
- [ ] Anonymous submission rules checked.
- [ ] AI disclosure checked if required.
- [ ] Supplement and code-release plan prepared.

## Rebuttal preparation

- [ ] Predicted reviewer objections listed.
- [ ] Extra experiments or clarifications prepared.
- [ ] Point-by-point response skeleton drafted.
